# 🔍 Site Structure Analyzer

Браузерный инструмент для анализа HTML-структуры веб-сайтов с выводом результатов в JSON.
Работает полностью на клиентской стороне (JavaScript), развёртывается на GitHub Pages.

## 📋 Что анализирует

| # | Раздел | Описание |
|---|--------|----------|
| 1 | **Кодировка** | UTF-8, windows-1251 и т.д. |
| 2 | **Главная страница и пагинация** | Тип пагинации (?page=N, /page/N/), примеры URL |
| 3 | **Поиск** | URL поиска, формы, параметры |
| 4 | **Сортировка** | Варианты сортировки (новые, популярные, рейтинг) |
| 5 | **Категории** | Список категорий/тегов с URL |
| 6 | **Карточки контента** | Селекторы названия, ссылки, превью, длительности, качества |
| 7 | **Страница просмотра** | Источники видео (.mp4, .m3u8), iframe-плееры, конфиги JS |
| 8 | **Похожий контент** | Блок "похожие видео" на странице просмотра |

## 🚀 Быстрый старт

### Вариант 1: Форк (самый быстрый)

1. Нажмите кнопку **Fork** в правом верхнем углу репозитория
2. Перейдите в **Settings → Pages**
3. Source: **Deploy from a branch**
4. Branch: **main** / **/ (root)**
5. Нажмите **Save**
6. Через 1-2 минуты сайт доступен по адресу: https://ВАШ_ЮЗЕРНЕЙМ.github.io/site-analyzer/

### Вариант 2: Ручное создание

```bash
# Клонируйте или создайте репозиторий
git clone https://github.com/YOUR_USERNAME/site-analyzer.git
cd site-analyzer

# Скопируйте файлы (index.html, style.css, parser.js, README.md)
# ... скопируйте файлы ...

# Запушьте
git add .
git commit -m "Initial commit: Site Structure Analyzer"
git push origin main

```
Затем включите GitHub Pages (см. раздел «Настройка GitHub Pages»).

### Вариант 3: Локальный запуск

# Любой статический сервер
npx serve .
# или
python -m http.server 8080
# или
php -S localhost:8080
Откройте http://localhost:8080 в браузере.

⚠️ Открытие index.html напрямую через file:// может не работать из-за CORS-политик браузера. Используйте локальный сервер.


# 📦 Виджет анализа видео-страниц (Video Page Analyzer)

## 🚀 Что нового?

В последнем обновлении добавлен мощный функционал для визуального анализа и отладки парсинга видео-контента. Реализованы две ключевые вкладки: **«Карточки»** (визуальный разбор превью) и **«Видео»** (детальный анализ источников контента).

---

## 🃏 Вкладка «Карточки»

Данный раздел отображает все спарсенные элементы (превью видео) в удобном для восприятия формате.

| Функция | Описание |
| :--- | :--- |
| **Сетка превью** | Все найденные карточки рендерятся визуально. Превью-изображения загружаются через прокси с реализованным `onerror`-фолбэком на случай недоступности картинки. |
| **Бейджи** | На каждом превью отображаются **длительность** видео и метка **качества** (HD / 4K), что повторяет поведение оригинального сайта. |
| **Название + ссылка** | Под изображением выводится название видео (обрезается до 2 строк) и кликабельная ссылка, ведущая на оригинальную страницу. |
| **Переключение вида** | Реализовано переключение между двумя режимами отображения: ▦ **Сетка** (Grid) и ☰ **Список** (List). |
| **Нумерация** | Каждая карточка автоматически получает порядковый номер (#1, #2, #3 ...), упрощающий навигацию. |
| **Шпаргалка селекторов** | В нижней части вкладки расположена таблица с CSS-селекторами, используемыми для парсинга каждого поля. Это значительно упрощает написание и отладку парсера. |
| **Empty state** | Если карточки на странице не найдены, отображается информативная заглушка вместо пустого экрана. |

---

## ▶️ Вкладка «Видео»

Раздел для технического анализа найденных видеоисточников и плееров.

| Функция | Описание |
| :--- | :--- |
| **Эмуляция плеера** | В зависимости от типа найденного источника:<br>• **MP4** — встраивается нативный `<video>` с контролами.<br>• **HLS** — отображается информативный плейсхолдер (так как требует специальной библиотеки).<br>• **iframe** — встраивается прямой iframe. |
| **Список источников** | Вывод всех найденных ссылок на медиафайлы (`.mp4`, `.m3u8`, `.webm`). Каждый тип сопровождается цветным бейджем, URL-адресом, а также кнопками **«Копировать»** и **«Открыть»**. |
| **iframe-плееры** | Выделенная секция для обнаруженных iframe-плееров (например, YouTube, Vimeo или сторонние встраиваемые плееры). |
| **JS-конфиги** | Если на странице обнаружены сниппеты JavaScript с конфигурацией плеера (настройки, пути к стримам), они выводятся в отдельный читаемый блок `<code>`. |
| **Мета-информация** | Отображается техническая информация: URL страницы, паттерн URL, использованные методы обнаружения, а также наличие блока «Похожие видео». |
| **Похожие видео** | Если на странице присутствуют ссылки на похожие ролики, они рендерятся в виде компактной сетки карточек со ссылками для быстрого перехода. |
| **Предупреждения** | Система выводит понятные предупреждения, если видео не найдено или загружается через iframe (что может ограничивать функционал парсинга). |

## 🛠 Технические изменения (parser.js)

Внесены следующие изменения в логику парсера для обеспечения работы нового интерфейса:

*   **`analyzeVideoCards`** — логика была переработана. Теперь парсит **все** карточки на странице, а не ограничивается первыми 5, что обеспечивает полноту данных на вкладке «Карточки».
*   **`analyzeVideoPage`** — метод расширен для извлечения `videoTitle` и `pageTitle`. Эти данные используются для отображения заголовка на вкладке видео.
*   **`showTab`** — исправлена передача аргументов (теперь передается `btnEl` вместо `event.target`), что гарантирует корректную работу переключения вкладок в строгом режиме.
*   **`copyToClipboard`** — добавлена новая утилита для кнопок «Копировать» на вкладке видео, обеспечивающая удобное копирование URL в буфер обмена.
*   **`setCardView`** — новая функция для динамического переключения режимов отображения (сетка/список) карточек без потери данных.
